# 경제 뉴스 서비스 PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트명
최신 경제소식 뉴스레터

### 1.2 프로젝트 목표
매일 아침 최신 경제 이슈를 크롤링하고 AI로 요약하여 모바일에서 편리하게 볼 수 있는 서비스

### 1.3 프로젝트 범위
- GitHub Pages에서 실행 가능한 정적 웹 서비스 (HTML, CSS, JS)
- GitHub Actions를 통한 자동 뉴스 크롤링 및 로컬 요약 생성
- 모바일 우선 반응형 웹 디자인
- 3개 탭 구조 (홈/뉴스목록/경제상식)
- RAG 기반 경제 상식 Q&A 시스템

## 2. 사용자 요구사항

### 2.1 주요 사용자
- 경제에 관심이 있는 한국인
- 경제 공부를 하고자 하는 사용자
- 아침 출근길과 저녁 퇴근길에 경제 정보를 확인하고 싶은 사용자

### 2.2 사용자 니즈
- 최신 경제 이슈 8개 확인 (오늘 + 어제)
- 모바일/데스크톱 모든 환경에서 접근 가능
- 모든 UI와 본문 내용은 한글로 구성
- 경제 상식에 대한 실시간 Q&A 기능

## 3. 기능 요구사항

### 3.1 홈 탭 (메인 화면)
- **오늘의 경제 이슈 요약**: 키워드 기반 자동 분석 및 요약
- **주요 관심사 표시**: 자동 추출된 키워드 (주식시장, 부동산, 금리정책 등)
- **뉴스 목록**: 최신 8개 뉴스 제목 표시
- **업데이트 시간**: 매일 아침 6시 자동 업데이트
- **새로고침 버튼**: 수동 업데이트 가능

### 3.2 뉴스 목록 탭
- **뉴스 목록**: 최신 경제 뉴스 8개 표시 (오늘 + 어제)
- **뉴스 소스**: 한국경제신문 (경제, 주식, 금융, 부동산, 산업, 글로벌, 정치)
- **뉴스 카드**: 제목, 출처, 발행시간, 요약
- **상세보기**: 뉴스 클릭 시 전체 본문 표시 (모달)
- **원문 링크**: 한국경제 원문으로 이동 가능
- **정렬**: 시간순 (최신순)

### 3.3 경제 상식 탭 (RAG Q&A 시스템)
- **실시간 Q&A**: 사용자 질문에 즉시 답변
- **제안 질문**: 자주 묻는 질문 10개 제공
- **카테고리별 지식**: 연금, 투자, 저축, 보험 등 10개 카테고리
- **관련 정보**: 답변과 함께 관련 지식 자동 제공
- **키워드 검색**: 질문 내용 기반 관련 정보 검색

### 3.4 데이터 관리
- **자동 크롤링**: GitHub Actions로 매일 06:00 KST 실행
- **로컬 요약**: OpenAI API 없이도 작동하는 로컬 요약 시스템
- **데이터 저장**: JSON 파일로 저장 (news.json, finance-tips.json)
- **폴백 시스템**: 크롤링 실패 시 샘플 데이터 자동 생성

## 4. 비기능 요구사항

### 4.1 성능 요구사항
- **페이지 로딩 시간**: 3초 이내
- **모바일 환경**: 부드러운 스크롤 및 터치 반응
- **오프라인 지원**: 기본 UI 동작 가능
- **API 독립성**: 외부 API 없이도 완전 동작

### 4.2 호환성 요구사항
- **모바일 브라우저**: Chrome, Safari, Samsung Internet
- **데스크톱 브라우저**: Chrome, Firefox, Edge
- **반응형 디자인**: 320px ~ 1920px 지원
- **GitHub Pages**: 정적 파일 기반 완전 호환

### 4.3 접근성 요구사항
- **WCAG 2.1 AA 수준**: 기본 접근성 준수
- **키보드 네비게이션**: 탭 키로 모든 기능 접근 가능
- **스크린 리더**: 기본적인 호환성

### 4.4 보안 요구사항
- **API 키 관리**: GitHub Secrets로 안전하게 관리
- **크롤링 윤리**: 적절한 User-Agent 및 지연시간 설정
- **데이터 보호**: 개인정보 수집 없음

## 5. 기술 스택

### 5.1 프론트엔드
- **HTML5**: 시맨틱 마크업
- **CSS3**: Flexbox, Grid, 애니메이션, 그라데이션
- **Bootstrap 5**: 반응형 컴포넌트
- **Vanilla JavaScript**: ES6+ 문법, 모듈화

### 5.2 백엔드/자동화
- **GitHub Actions**: 자동 크롤링 워크플로우
- **Python 3.9+**: 크롤링 및 데이터 처리
- **BeautifulSoup4**: HTML 파싱
- **로컬 요약 시스템**: 키워드 기반 자동 요약

### 5.3 데이터 소스
- **한국경제신문**: 메인 경제 뉴스 소스
- **다중 섹션**: 경제, 주식, 금융, 부동산, 산업, 글로벌, 정치
- **아카이브 크롤링**: 오늘 + 어제 뉴스 수집

### 5.4 RAG 시스템
- **정적 데이터**: JavaScript 기반 경제 상식 데이터베이스
- **키워드 검색**: 질문 기반 관련 정보 검색
- **실시간 답변**: 서버 없이도 즉시 Q&A 제공

## 6. UI/UX 요구사항

### 6.1 디자인 원칙
- **모던한 디자인**: 그라데이션과 카드형 레이아웃
- **일관성**: 통일된 색상 팔레트와 타이포그래피
- **가독성**: 명확한 계층 구조와 충분한 여백
- **모바일 우선**: 터치 친화적 인터페이스

### 6.2 레이아웃 구조
- **하단 고정 탭바**: 3개 탭 (홈/뉴스/경제상식)
- **상단 헤더**: 날짜 및 새로고침 버튼
- **메인 콘텐츠**: 카드 기반 리스트
- **모달/오버레이**: 상세보기 및 Q&A 답변

### 6.3 색상 팔레트
- **주 색상**: 블루 그라데이션 (#667eea → #764ba2)
- **보조 색상**: 화이트, 그레이 톤
- **강조 색상**: 오렌지 (#ff6b6b)
- **RAG 색상**: 보라 그라데이션 (#667eea → #764ba2)

## 7. 사용자 여정

### 7.1 아침 출근길 (06:00-09:00)
1. **앱 실행** → 홈 탭에서 오늘의 경제 이슈 요약 확인
2. **주요 관심사 파악** → 키워드 기반 주요 이슈 확인
3. **관심 뉴스 상세보기** → 뉴스 탭으로 이동하여 상세 확인
4. **경제 상식 학습** → 경제상식 탭에서 Q&A 활용

### 7.2 저녁 퇴근길 (18:00-21:00)
1. **하루 뉴스 정리** → 뉴스 탭에서 8개 뉴스 전체 확인
2. **중요 뉴스 상세보기** → 관심 있는 뉴스 전체 내용 파악
3. **경제 상식 복습** → Q&A로 금융상품 정보 학습
4. **내일 준비** → 다음날 관심 키워드 파악

## 8. 성공 지표

### 8.1 기능적 성공 지표
- ✅ 매일 06:00 KST 자동 크롤링 정상 동작
- ✅ 로컬 요약 시스템 정상 동작 (API 독립)
- ✅ 3개 탭 전환 기능 정상 동작
- ✅ RAG Q&A 시스템 정상 동작
- ✅ 반응형 디자인 모든 해상도에서 정상

### 8.2 사용성 성공 지표
- ✅ 사용자가 10초 내에 오늘의 경제 이슈 파악 가능
- ✅ 모바일 환경에서 터치 조작 문제 없음
- ✅ 뉴스 상세보기 로딩 시간 2초 이내
- ✅ Q&A 답변 시간 1초 이내

### 8.3 콘텐츠 품질 지표
- ✅ 뉴스 요약 정확도 90% 이상 (키워드 기반)
- ✅ 경제 상식 콘텐츠 완성도 95% 이상
- ✅ 크롤링 데이터 신선도 (48시간 이내)
- ✅ 뉴스 다양성 (8개 뉴스, 2일간 커버리지)

## 9. 배포 및 운영

### 9.1 배포 방식
- **GitHub Pages**: 정적 파일 기반 무료 호스팅
- **자동 배포**: 코드 푸시 시 자동 배포
- **CDN 활용**: 빠른 전 세계 접근

### 9.2 운영 방식
- **무인 운영**: GitHub Actions로 완전 자동화
- **비용 절감**: 외부 API 의존성 최소화
- **확장성**: 필요시 추가 뉴스 소스 쉽게 추가 가능

### 9.3 모니터링
- **크롤링 상태**: GitHub Actions 로그 확인
- **사용자 피드백**: GitHub Issues 활용
- **성능 모니터링**: 브라우저 개발자 도구 활용
